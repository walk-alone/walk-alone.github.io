常见设计模式

创建型模式

简单工厂模式（Simple Factory）简易计算器ok
工厂方法模式（Factory Method）雷锋工厂 ok
抽象工厂模式（Abstract Factory） 供一个创建一系列相关或相互依赖对象的接口，而无需指定它们的类 @@
建造者模式（Builder）  “画”出一个四肢健全（头身手腿）的小人
原型模式（Prototype）  return this  ok
单例模式（Singleton）


结构型模式

适配器模式(Adapter)  将一个类的接口转换成为客户希望的另外一个接口
桥接模式(Bridge)
组合模式(Composite)
装饰模式(Decorator)   一个人穿衣服(可多件)
外观模式(Facade)  接口将几种调用分别组合成为两组，用户通过接口调用其中的一组
享元模式(Flyweight)
代理模式(Proxy)


行为型模式

职责链模式(Chain of Responsibility)
命令模式(Command)
解释器模式(Interpreter)
迭代器模式(Iterator)
中介者模式(Mediator)
备忘录模式(Memento)
观察者模式(Observer)  让多个观察对象同时监听一个主题对象，当主题对象状态发生变化时会通知所有观察者。
状态模式(State)       当一个对象的内在状态改变时允许改变其行为，这个对象看起来像是改变了其类
策略模式(Strategy)    超市售卖策略ok
模板方法模式(Template Method) 考试时使用同一种考卷（父类），不同学生上交自己填写的试卷（子类方法的实现）
访问者模式(Visitor)
